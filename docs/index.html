TypeScript-first Caching Framework

fusion-cache

Modern, extensible, driver-based caching for Node.js and Bun. Zero runtime dependencies. Beautiful developer experience.

Get Started API Reference
🛡️

TypeScript First

Full type safety with generics, IntelliSense, and strict typing throughout.

📦

No runtime dependencies. Optional peer deps for YAML/TOML formats only.

"5s", "30m", "2h", "1d", "1w" — no more raw milliseconds.

ESM + CJS

Async-First

Promise-based API throughout. Perfect for modern async/await workflows.

# or
pnpm add fusion-cache

bash

Quick Start

TypeScript 
import { FusionCache, MemoryDriver } from 'fusion-cache';

const cache = new FusionCache({
  driver: new MemoryDriver(),
});

// Store with human-readable TTL
await cache.set('user:1', { id: 1, name: 'Alice' }, { ttl: '30m' });

// Retrieve typed values
const user = await cache.get<{ id: number; name: string }>('user:1');

// Check existence
if (await cache.has('user:1')) {
  console.log('User cached');
}

// Delete
await cache.delete('user:1');

// Clear all
await cache.clear();

fusion-cache supports human-readable TTL strings alongside raw millisecond values:

"30m"30 minutes1,800,000
"2h"2 hours7,200,000
"1d"1 day86,400,000
"1w"1 week604,800,000
5000 (number)5 seconds5,000
0No expiration—
TypeScript 
// Human-readable
await cache.set('key', value, { ttl: '5m' });

// Raw milliseconds
await cache.set('key', value, { ttl: 300000 });

// No expiration
await cache.set('key', value, { ttl: 0 });

// Parse TTL manually
import { parseTTL } from 'fusion-cache';
parseTTL('30m'); // → 1800000

Namespacing

Use the prefix option to namespace keys. All keys stored through a prefixed cache are automatically prefixed:

TypeScript 
const userCache = new FusionCache({ driver, prefix: 'users' });
const postCache = new FusionCache({ driver, prefix: 'posts' });

await userCache.set('1', userData);    // Stored as "users:1"
await postCache.set('1', postData);    // Stored as "posts:1"

// Different caches, same key, no collision
await userCache.get('1'); // → userData
await postCache.get('1'); // → postData

Cache-Aside Pattern

The getOrSet method implements the cache-aside pattern. It returns the cached value if it exists, otherwise calls your factory function, stores the result, and returns it:

TypeScript 
const user = await cache.getOrSet(
  `user:${id}`,
  async () => await db.users.findById(id),
  { ttl: '5m' }
);
How it works: The factory function is only called on a cache miss. On subsequent calls with the same key, the cached value is returned without invoking the factory.

MemoryDriver

In-memory cache using Map. Zero configuration. Fastest option for hot data.

TypeScript 
import { MemoryDriver } from 'fusion-cache';

const driver = new MemoryDriver({
  maxSize: 1000,       // Max entries (0 = unlimited, default: 0)
  checkInterval: 60000 // Cleanup interval in ms (0 = no periodic cleanup)
});

Options

OptionTypeDefaultDescription
maxSizenumber0Maximum entries. FIFO eviction when exceeded. 0 = unlimited.
checkIntervalnumber0Periodic cleanup interval in ms. 0 = no periodic cleanup.

Behavior

  • TTL Expiration: Lazy — entries are checked on access
  • Eviction: FIFO when maxSize is exceeded
  • Persistence: None — data lost on process exit
  • Best for: Hot data, L1 cache, session data

FileDriver

Persists cache to the filesystem. Write-through with in-memory cache layer. Supports JSON, YAML, and TOML formats.

TypeScript 
import { FileDriver } from 'fusion-cache';

const driver = new FileDriver({
  path: '.cache',        // Directory path (default: '.cache')
  format: 'json',        // 'json' | 'yaml' | 'toml'
  flushInterval: 1000    // Debounce interval in ms (0 = flush on disconnect only)
});

await driver.connect();  // Creates directory and loads existing cache

Options

OptionTypeDefaultDescription
pathstring".cache"Directory path for cache files.
format"json" | "yaml" | "toml""json"File format for persistence.
flushIntervalnumber1000Debounce interval in ms. 0 = flush only on disconnect.

Format Requirements

FormatDependencyInstall
JSONNone (built-in)—
YAMLyamlpnpm add yaml
TOMLsmol-tomlpnpm add smol-toml

Behavior

  • Write-through: In-memory cache backed by filesystem
  • Atomic writes: Uses tmp file + rename for crash safety
  • Debounced flush: Batches I/O operations
  • Long keys: Automatically hashed (SHA-256) when exceeding 200 chars
  • Best for: Persistent config, L2 cache, cross-restart data
Production tip: Use MemoryDriver for L1 (hot data) + FileDriver for L2 (persistence). The FileDriver's in-memory cache layer provides fast reads while the filesystem provides durability.

Custom Drivers

Implement the CacheDriver interface to create your own backend:

TypeScript 
import type { CacheDriver, CacheSetOptions } from 'fusion-cache';

class RedisDriver implements CacheDriver {
  async get<T>(key: string): Promise<T | null> {
    // Your implementation
  }

  async set<T>(key: string, value: T, options?: CacheSetOptions): Promise<void> {
    // Your implementation
  }

  async delete(key: string): Promise<boolean> {
    // Your implementation
  }

  async has(key: string): Promise<boolean> {
    // Your implementation
  }

  async clear(): Promise<void> {
    // Your implementation
  }

  // Optional methods
  async keys(pattern?: string): Promise<string[]> {
    // Your implementation
  }

  async connect(): Promise<void> {
    // Initialize connection
  }

  async disconnect(): Promise<void> {
    // Clean up connection
  }
}

Interface Contract

MethodRequiredDescription
get<T>(key)YesReturn cached value or null
set<T>(key, value, options?)YesStore value with optional TTL
delete(key)YesRemove key, return true if existed
has(key)YesCheck if key exists (not expired)
clear()YesRemove all entries
keys(pattern?)NoList keys with optional * wildcard
connect()NoInitialize driver
disconnect()NoClean up resources

TypeScript Types

TypeScript 
// Human-readable TTL format
type HumanTTL =
  | `${number}s`  // seconds
  | `${number}m`  // minutes
  | `${number}h`  // hours
  | `${number}d`  // days
  | `${number}w`; // weeks

// Cache driver interface
interface CacheDriver {
  connect?(): Promise<void>;
  disconnect?(): Promise<void>;
  get<T>(key: string): Promise<T | null>;
  set<T>(key: string, value: T, options?: CacheSetOptions): Promise<void>;
  delete(key: string): Promise<boolean>;
  has(key: string): Promise<boolean>;
  clear(): Promise<void>;
  keys?(pattern?: string): Promise<string[]>;
}

// Set options
interface CacheSetOptions {
  ttl?: number | HumanTTL;
  tags?: string[];
}

// Cache constructor options
interface FusionCacheOptions {
  driver: CacheDriver;
  prefix?: string;
  defaultTTL?: number | HumanTTL;
}

// MemoryDriver options
interface MemoryDriverOptions {
  maxSize?: number;
  checkInterval?: number;
}

// FileDriver options
type FileFormat = 'json' | 'yaml' | 'toml';

interface FileDriverOptions {
  path?: string;
  format?: FileFormat;
  flushInterval?: number;
}

Error Handling

fusion-cache provides a typed error hierarchy:

FusionCacheError Base error for all fusion-cache errors
InvalidTTLInputError Invalid TTL format (e.g., "abc")
DriverConnectionError Driver failed to connect
DriverNotSupportedError Driver doesn't support a required operation
FileDriverError FileDriver-specific errors (missing deps, I/O)
TypeScript 
import {
  FusionCacheError,
  InvalidTTLInputError,
  DriverConnectionError,
  FileDriverError,
} from 'fusion-cache';

try {
  await cache.set('key', 'value', { ttl: 'invalid' });
} catch (error) {
  if (error instanceof InvalidTTLInputError) {
    console.error('Bad TTL format');
  }
}
